Lekcja III - HTML 2

Nowe znaczniki:

  • <br/> - nowa linia (nie dublujemy w celu uzyskania większego odstępu!)
  • <b> </b> - pogrubienie tekstu (nie ma znaczenia semantycznego
  • <i> </i> - kursywa (nie ma znaczenia semantycznego)
  • <strong> </strong> - pogrubienie, ale dodatkowo podkreśla wysoką istotność frazy
  • <em> </em> - podkreśla, że tekst może mieć inne znaczenie (np. ironia)
  • <a href="tutaj ścieżka dostępu">nazwa hiperłącza
  • </a> - linki/łącza wewnątrz dokumentu
  • <element id="moj-identyfikator"> - atrybut id identyfikuje element
  • <nav> - nawigacja
  • <img src="ścieżka dostępu" alt="tekst alternatywny"/> - wstaw obrazek. Inne zamknięcie!

WAŻNE!!!

  1. Jeżeli w <a > </a> odwołujemy się do id przed nazwą musi być #
  2. Dodatkowy <header> </header> - w "body" ale przed "main" (Wpisujemy tam elementy, które chcemy powtarzać na wszystkich stronach. Jeżeli stopka ma być powtarzana to przenosimy ją za "main" ale w "body")

Lekcja V - CSS II

Styl przy znaczniku z wykorzystaniem atrybutu style nazywamy stylem inline.

  1. <znacznik style="color:yellow;">treść</znacznik> ---> ogólnie nie polecam
  2. <znacznik class="name">treść</znacznik> nazwa zapisana kebab-case
  3. <znacznik id="name">treść</znacznik> nazwa zapisana kebab-case

W css odwołujemy się do klasy za pomocą .name a do id za pomocą #id. ID używaj tylko do jednego elementu na stronie.
Zmiana koloru linku w nav: nav a{color:} Pamiętaj aby nie korzystać z !important - ponieważ nadpisuje styl (nie zwraca uwagi na inne właściwości)

Specyficzność:

  1. !important
  2. style=""
  3. #name
  4. .name
  5. element

Jeżeli specyficzności się równoważą to wygrywa styl zapisany niżej. Ogólnie unikamy zagnieżdżenia.

Lekcja VI - Jak działa Internet i przeglądarka

HTTP - protokół, którego używa przeglądarka (protokół do transportowania htmla + reszta). HTTPS - jest szyfrowany, obecnie rekomendowany.
URL - adres jaki wpisujemy

https://cotenfrontend.pl:443/index.html?parametr=wartość

  1. https:// - protokół
  2. cotenfrontend.pl - host(adres serwera)
  3. :443 - port (nie widzimy, przeglądarka automatycznie to dopisuje)- na którym porcie na serwerze znajdują się pliki
  4. / - super ważny! (root) ścieżki względne i bezwzględne - lepiej stosuj: /img/... (druga z wymienionych w prezentacji)
  5. index.html - ścieżka dostępu do zasobu
  6. ?parametr=wartość - parametry

Wpisuje adres www -> przeglądarka szuka serwera [DNS - tlumaczy nazwy pisane na adresy IP] -> Po znalezieniu serwera odpytuje go o index -> serwer wysyła zawartość html do przeglądarki -> przeglądarka wczytuje i szuka dodatkowych zasobów (obrazki, style itp.) i dla każdego z zasobów wykonuje zapytanie - > przeglądarka sprawdza składnie strony i ją generuje.

KODY ODPOWIEDZI HTTP:
- 200 - jest ok
- 30x - szukasz w złym miejscu - przekieruję Cię w dobre (bo wiem gdzie)
- 404 - nie mam tego, czego potrzebujesz... Sorry
- 50x - błędy wewnętrzne serwera (nie możemy z tym nic zrobić)

Lekcja VII - JavaScript

Pamiętaj że:

Java != JavaScript

Uczymy się ECMAScript 2015+ (ES6+) - jest to ustandaryzowany przez organizację ECMA skryptowy język programowania. JS implementuje rzeczy ze specyfikacji ECMAScript. ES6 wprowadza wiele udogodnień. Nowe feature'y mogą nie być wspierane we wszystkich przeglądarkach, ale są na to sposoby.

JS składa sie z:

  • wartości - istniejący zbiór, niezmienialne
  • operacji porównywania
  • literałów
  • zmiennych
  • obiektów
  • funkcji

Mają swój znacznik <script> Pierwszy jaki poznamy to alert(''); - wyświetla komunikat po załadowaniu strony (pamiętaj o ciapkach!). Drugi to console.log('') - wyświetla tekst w konsoli.
Ogólnie tekst wypisujemy w ciapkach pojedynczych 'a zmienne bez niczego. Jeżeli chcemy łaczyć te dwie rzeczy to stosujemy ~tekst ${zmienna}~
